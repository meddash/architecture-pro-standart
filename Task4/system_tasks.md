# Крупные задачи по системам

## Сервис ставок

1. Реализовать единое хранилище актуальных депозитных ставок с версиями и датой публикации.
2. Реализовать ролевую модель для сотрудников бэк-офиса депозитов и кредитов.
3. Реализовать компонент публикации ставок для внутренних и внешних потребителей.
4. Реализовать журнал изменений ставок и аудит действий пользователей.

## Система внутреннего кол-центра

1. Добавить экран или виджет просмотра актуальных депозитных ставок.
2. Реализовать интеграцию с внутренним API/витриной сервиса ставок.
3. Добавить отображение версии и времени актуальности ставок для оператора.
4. Обеспечить обработку ошибок и fallback-сценарии при недоступности сервиса ставок.

## Экспорт ставок для партнёра

1. Реализовать генерацию файлов выгрузки ставок в согласованном формате.
2. Реализовать расписание формирования файлов и механизм повторной отправки.
3. Реализовать журнал выгрузок с версиями файлов и статусами доставки.
4. Реализовать ограничение состава данных для внешнего контура.

## Файловый канал обмена / внешний контур партнёра

1. Согласовать формат файла, регламент приёма и правила обработки ошибок.
2. Настроить операционный процесс передачи и подтверждения доставки файла.
3. Реализовать импорт ставок во внешнюю систему партнёрского кол-центра.
4. Реализовать отображение актуальных ставок сотрудникам партнёра.

## Мониторинг и сопровождение

1. Настроить мониторинг успешности публикации ставок и доставки файлов.
2. Настроить оповещения по сбоям формирования, передачи и импорта файлов.
3. Подготовить регламент ручного повтора отправки и эскалации инцидентов.
4. Подготовить эксплуатационную документацию по поддержке нового контура.

## Организационные задачи

1. Назначить владельца справочника ставок и регламент публикации.
2. Согласовать SLA актуальности ставок для внутреннего и партнёрского кол-центров.
3. Провести обучение сотрудников внутреннего и партнёрского кол-центров.
4. Подготовить пилот и критерии готовности к масштабированию на период маркетинговой кампании.

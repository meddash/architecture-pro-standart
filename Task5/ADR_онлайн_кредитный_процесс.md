### <a name="_qmphm5d6rvi3"></a>**Решение**
Для онлайн-кредитного процесса предлагается добавить отдельный прикладной слой вокруг существующих систем кредитного контура, а не расширять прямую роль АБС в цифровом процессе. Ключевыми элементами решения становятся сервис кредитных заявок, сервис предложений, сервис уведомлений и интеграционный слой к кредитному конвейеру и АБС.

Логика выбора решения:

1. АБС перегружена и не может быть центральной онлайн-точкой входа для цифровых кредитных заявок.
2. Обмен АБС с кредитным конвейером ускорить нельзя, поэтому для онлайн-каналов нужен отдельный путь доставки заявок в кредитный конвейер.
3. Для нового клиента решение по сайту должно по возможности выдаваться сразу после скоринга, а для клиента интернет-банка нужно использовать предодобренные предложения и упрощённый повторный скоринг.
4. Сервис кредитных заявок должен стать единым реестром цифровых заявок и статусов, чтобы связать сайт, интернет-банк, отделение, кредитный конвейер и уведомления.
5. Показ предодобренных предложений и обработка same-day требуют выделить отдельную read-модель или сервис предложений, а не читать их напрямую из АБС.

Состав решения:

- Сайт банка: внешний канал для новых клиентов, позволяет подавать кредитные заявки и получать предварительное решение.
- Интернет-банк: канал для действующих клиентов, показывает кредитные продукты, предодобренные предложения и позволяет отправить заявку с СМС-подтверждением.
- Сервис кредитных заявок: принимает заявки из цифровых каналов, хранит статусы, маршрутизирует обработку, связывает клиента и заявку между каналами.
- Сервис предложений: хранит и публикует предодобренные предложения и read-модель условий по кредитным продуктам.
- Скоринговый адаптер: контролирует вызовы в систему скоринга и применяет правила, чтобы не перегружать её лишними запросами.
- Интеграционный слой к кредитному конвейеру: передаёт цифровые заявки напрямую в кредитный конвейер и получает статусы обработки.
- Интеграционный слой к АБС: используется для продолжения заявки в отделении, для клиентских данных и для последующих операций после финального решения.
- Сервис уведомлений: отправляет клиенту СМС по ключевым изменениям статуса заявки.

Ключевое архитектурное решение:

- Цифровые заявки из сайта и интернет-банка должны попадать в сервис кредитных заявок и далее в кредитный конвейер напрямую, минуя суточный маршрут АБС → кредитный конвейер.
- АБС остаётся важной системой банковского ядра, но не центральной системой оркестрации онлайн-кредитного процесса.
- Для отделения обеспечивается сквозной идентификатор заявки, чтобы сотрудник мог продолжить уже начатый процесс.

### <a name="_bjrr7veeh80c"></a>**Альтернативы**
Опишите здесь наиболее важные альтернативные решения.

1. Оставить АБС единственной точкой входа для всех кредитных заявок, в том числе онлайн.
Минусы: усиливается нагрузка на уже перегруженную АБС, сохраняется зависимость от суточной передачи в кредитный конвейер, same-day сценарий для предодобренных заявок остаётся нерешённым.

2. Передавать цифровые заявки в кредитный конвейер только через существующую интеграцию по БД из АБС.
Минусы: технически не позволяет обрабатывать часть заявок в тот же рабочий день и ломает целевое ожидание по скорости.

3. Выполнять полный скоринг по каждому просмотру предложения или массово пересчитывать предложения заранее.
Минусы: создаёт лишнюю нагрузку на систему кредитного скоринга, что прямо противоречит ограничениям.

4. Полностью переписать кредитный процесс на новую микросервисную платформу в рамках одного этапа.
Минусы: слишком высокий объём изменений для горизонта одного года, большие проектные риски и зависимость от модернизации большого числа систем.

**Недостатки, ограничения, риски**

Подробно опишите здесь недостатки, ограничения и риски выбранного решения.

1. В архитектуре появляется новый прикладной слой, который нужно сопровождать: сервис кредитных заявок, сервис предложений, интеграционные адаптеры, уведомления.
2. АБС полностью не исключается из процесса: для отделения, клиентских данных и банковских операций зависимость от неё остаётся.
3. При расхождении моделей данных между сервисом кредитных заявок, кредитным конвейером и АБС возникает риск потери согласованности и дублирования идентификаторов.
4. Если правила вызова скоринга будут определены плохо, можно либо перегрузить скоринг, либо получить слишком консервативный пользовательский опыт.
5. Предодобренные предложения требуют отдельного процесса актуализации; иначе в интернет-банке клиент будет видеть устаревшие предложения.
6. Для новых клиентов сохраняется ограничение по идентификации в отделении, поэтому полностью бесшовный цифровой процесс для этого сегмента не достигается.
